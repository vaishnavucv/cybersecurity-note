# Advanced Cybersecurity and Ethical Hacking L1 Career Track

**Trainer:** Vaishnavu C V  
**Course level:** Beginner to L1 cybersecurity readiness  
**Delivery mode:** Live online with cyberrange  

## Course Overview

This course is a practical cybersecurity career starter program designed to help learners build L1 level security analyst capability through structured theory, guided labs, cyberrange practice, AWS cloud security exposure, report writing, and a final capstone project.

The course is built for learners who want more than theory. Students will operate tools, investigate logs, validate vulnerabilities in authorized labs, document findings, understand remediation, and present evidence in a professional format.

**Positioning statement:**  
From zero cybersecurity background to practical L1 readiness through live cyberrange labs, cloud security practice, daily exercises, and mentor guided reporting.

## Important Legal and Ethical Scope

This course is strictly designed for authorized learning environments only. All practical activities are limited to approved cyberrange machines, owned systems, assigned vulnerable applications, training labs, and controlled cloud accounts.

Students must not test, scan, attack, or attempt to access public websites, third party systems, public IP addresses, wireless networks, cloud assets, user accounts, or organization assets without written permission and approved scope.

### Not permitted in this course

1. Public target hacking.
2. Credential theft against real users.
3. Real malware deployment.
4. Stealth, persistence, or evasion activity on third party systems.
5. Unauthorized wireless testing.
6. Testing outside the assigned cyberrange, lab, or approved cloud account.
7. Bypassing legal, academic, platform, or organizational controls.

The objective is to build defensive, ethical, professional, and employable cybersecurity capability.

## What Students Will Learn

By the end of the program, a serious student is expected to be able to:

1. Set up and operate a basic cybersecurity lab using Kali Linux, Ubuntu, Windows, vulnerable applications, SIEM, and approved cloud resources.
2. Understand cybersecurity foundations, cyber threats, ethics, risk, and security best practices.
3. Use Linux and Windows fundamentals for basic security investigation.
4. Understand networking, ports, protocols, packet analysis, and basic network security concepts.
5. Perform safe reconnaissance and asset enumeration inside approved labs.
6. Validate common lab vulnerabilities and write professional remediation notes.
7. Use Burp Suite or OWASP ZAP for guided web application security testing in lab environments.
8. Understand authentication, access control, injection, file upload risk, misconfiguration, API risk, and secure logging concepts.
9. Triage SIEM alerts, inspect Linux, Windows, web, and network logs, and raise clean incident tickets.
10. Understand basic incident response, digital evidence handling, timeline building, containment, and recovery notes.
11. Understand AWS security basics such as IAM, CloudTrail, GuardDuty, Security Hub, Config, S3 exposure, encryption, and least privilege.
12. Build a beginner portfolio with reports, screenshots, risk ratings, remediation guidance, and a final capstone presentation.

## Detailed Topic Coverage

### Cybersecurity Foundation

1. Definition, importance, and scope of cybersecurity.
2. Types of cyber threats and basic attack lifecycle awareness.
3. Cybersecurity ethics, legal scope, and professional conduct.
4. Security principles, defense in depth, least privilege, and risk based thinking.
5. Security roles such as SOC L1, Security Analyst, VAPT Intern, Cloud Security beginner, and DevSecOps beginner.

### Linux for Security

1. Basic Linux commands.
2. File system and directory structure.
3. Users, groups, permissions, and ownership.
4. Networking commands.
5. Services, processes, and log review.
6. Basic hardening checks.

### Windows Security Basics

1. Access control, permissions, and UAC.
2. Windows Firewall and Microsoft Defender awareness.
3. Event Viewer basics.
4. CMD and PowerShell for administration evidence.
5. User, process, service, and log review.
6. Basic endpoint security checklist.

### Networking for Security

1. Types of networks and common architecture patterns.
2. Ports, protocols, IP addressing, and subnetting.
3. DNS, HTTP, HTTPS, SSH, RDP, SMB, and basic service exposure.
4. Packet capture review using approved sample traffic.
5. Network mapping and basic segmentation awareness.

### Ethical Hacking and Vulnerability Assessment

1. Scope and authorization.
2. Passive reconnaissance inside the approved training profile.
3. Active enumeration inside the cyberrange.
4. Service discovery and version review.
5. Vulnerability validation in lab assets.
6. Severity rating and business impact explanation.
7. Remediation writing and report formatting.
8. Password hashing and authentication security awareness.
9. Active Directory security overview, based on batch level.
10. Wireless security awareness, only in approved lab scope when included.

### Web Application and API Security

1. HTTP fundamentals and web architecture.
2. Proxy based testing workflow using Burp Suite or OWASP ZAP.
3. Directory and endpoint mapping in lab applications.
4. Authentication and session weaknesses.
5. Broken access control.
6. File upload risk.
7. Cross site scripting awareness.
8. Injection and input validation awareness.
9. Security misconfiguration and exposed secrets.
10. API security fundamentals.
11. Secure logging and alerting gaps.
12. OWASP Top 10 and OWASP API Security awareness.

### SIEM, SOC, and Detection

1. SOC workflow and alert triage.
2. Linux, Windows, web, and network log review.
3. SIEM dashboard basics.
4. Suspicious login investigation.
5. Service scan and web error pattern review.
6. Incident ticket writing.
7. Severity, impact, owner, next action, and closure notes.

### Incident Response and Digital Evidence

1. Incident response lifecycle awareness.
2. Evidence capture and chain of custody basics.
3. Timeline creation from logs.
4. Containment, recovery, and lessons learned notes.
5. Basic digital forensics awareness using safe lab evidence.

### AWS Cloud Security Basics

1. AWS shared responsibility model.
2. IAM users, groups, roles, policies, and least privilege.
3. CloudTrail and CloudWatch logging.
4. AWS Config visibility.
5. GuardDuty findings.
6. Security Hub posture review.
7. IAM Access Analyzer.
8. S3 public exposure prevention.
9. KMS encryption basics.
10. VPC, security groups, and NACL awareness.

### Responsible AI in Cybersecurity

1. AI assisted checklist creation.
2. AI assisted report drafting after manual verification.
